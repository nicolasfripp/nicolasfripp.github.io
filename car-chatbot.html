Car Chatbot

Búsqueda Semántica de Vehículos mediante Lenguaje Natural

Objetivo del Proyecto

Desarrollar una aplicación conversacional que permita a usuarios buscar vehículos mediante lenguaje natural, superando las limitaciones de los filtros tradicionales de búsqueda. El sistema debía ser capaz de interpretar consultas complejas como "necesito un SUV rojo por menos de $10,000" y devolver resultados relevantes con documentación de fuentes.

Problema Abordado

Los sistemas tradicionales de búsqueda de vehículos requieren que el usuario conozca y utilice filtros exactos (marca, modelo, año, precio). Esto limita la experiencia cuando el usuario tiene una idea general de lo que busca pero no conoce los parámetros específicos. Este proyecto resuelve el problema permitiendo búsquedas en lenguaje natural sobre un dataset de 38,531 vehículos con 30 atributos cada uno.

Decisiones Técnicas

  • Se eligió RAG (Retrieval Augmented Generation) sobre filtrado tradicional para permitir consultas semánticas flexibles
  • FAISS (Facebook AI Similarity Search) para indexación eficiente de 38,531 embeddings vectoriales
  • GPT-4o-mini como modelo de lenguaje por su balance costo/rendimiento (~$0.0001 USD por consulta)
  • Implementación de un SummaryBufferMemory custom que mantiene los últimos 1,000 tokens verbatim y resume automáticamente el historial anterior
  • Arquitectura desacoplada: backend FastAPI + frontend React comunicados via REST API

Tecnologías y Metodologías

Backend

FastAPI Python 3.11+ LangChain FAISS OpenAI API

Frontend

React 19 Vite Axios CSS3

Metodología

RAG Pipeline Embeddings Semánticos Memoria Conversacional

Datos

Dataset de 38,531 vehículos con 30 atributos (marca, modelo, año, precio, color, tipo de carrocería, combustible, etc.)

Arquitectura del Sistema

  1. El usuario envía una consulta en lenguaje natural
  2. La consulta se convierte en un vector de embeddings usando text-embedding-3-small de OpenAI
  3. FAISS busca los 5 vehículos más similares semánticamente en el índice vectorial
  4. Los datos de los vehículos recuperados se formatean como contexto
  5. GPT-4o-mini genera una respuesta en lenguaje natural usando el contexto + historial de conversación
  6. El SummaryBufferMemory actualiza el historial, comprimiendo mensajes antiguos automáticamente

Resultados Obtenidos

  • Tiempo de respuesta: 1-2 segundos por consulta
  • Costo por consulta: ~$0.0001 USD (altamente económico)
  • Soporte bilingüe: consultas en español e inglés
  • Interfaz responsive con indicador de typing en tiempo real y cards de fuentes expandibles
  • Capacidad de manejar preguntas de seguimiento con contexto ("¿Cuál es el más barato de los que me mostraste?")
  • Documentación completa de fuentes: cada respuesta puede mostrar los vehículos utilizados como referencia
